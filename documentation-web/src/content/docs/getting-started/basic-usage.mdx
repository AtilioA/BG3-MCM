# Getting started with MCM

This guide walks you through integrating MCM into your BG3SE mod. The process is simple: create a blueprint JSON file and use the MCM API in your Lua code.

<Aside type="tip" title="Complete examples available">
  Check out these working MCM integrations: - [Auto Send Food To
  Camp](https://github.com/AtilioA/BG3-auto-send-food-to-camp) - [Smart
  Autosaving](https://github.com/AtilioA/BG3-smart-autosaving)
</Aside>

## Overview

Integrating MCM into your mod involves three main steps:

<Steps>
  1. **Create an `MCM_blueprint.json`** file defining your settings
  2. **Add MCM as a dependency** in your `meta.lsx`
  3. **Use `MCM.Get()`** to read settings in your Lua code
</Steps>

## Step 1: Create the blueprint

Create a file named `MCM_blueprint.json` alongside your `meta.lsx`:

```json
{
  "SchemaVersion": 1,
  "ModName": "My Awesome Mod",
  "Tabs": [
    {
      "TabId": "general",
      "TabName": "General",
      "Settings": [
        {
          "Id": "enable_mod",
          "Name": "Enable Mod",
          "Type": "checkbox",
          "Default": true,
          "Tooltip": "Master toggle for the mod"
        },
        {
          "Id": "intensity",
          "Name": "Intensity",
          "Type": "slider_int",
          "Default": 50,
          "Options": {
            "Min": 0,
            "Max": 100
          },
          "Description": "Adjust the intensity level"
        }
      ]
    }
  ]
}
```

## Step 3: Use MCM in your code

Access settings anywhere in your Lua code:

```lua
-- Get a setting value
local enabled = MCM.Get("enable_mod")
local intensity = MCM.Get("intensity")

-- Set a setting value
MCM.Set("intensity", 75)

-- Listen for changes
Ext.ModEvents.BG3MCM["MCM_Setting_Saved"]:Subscribe(function(payload)
    if payload.modUUID ~= ModuleUUID then return end

    if payload.settingId == "enable_mod" then
        MyMod:SetEnabled(payload.value)
    end
end)
```

## Keybinding example

Define a keybinding in your blueprint:

```json
{
  "Id": "key_action",
  "Name": "Action Hotkey",
  "Type": "keybinding_v2",
  "Default": {
    "Keyboard": {
      "Key": "F",
      "ModifierKeys": ["LShift"]
    }
  },
  "Description": "Hotkey to trigger the action"
}
```

Register the callback in your client-side Lua:

```lua
MCM.Keybinding.SetCallback("key_action", function(e)
    MyMod:DoAction()
end)
```

## Event button example

Define an event button in your blueprint:

```json
{
  "Id": "reset_button",
  "Name": "Reset Settings",
  "Type": "event_button",
  "Options": {
    "ConfirmDialog": {
      "Title": "Confirm Reset",
      "Message": "Are you sure you want to reset all settings?",
      "ConfirmText": "Reset",
      "CancelText": "Cancel"
    },
    "Cooldown": 5
  },
  "Tooltip": "Click to reset mod"
}
```

Register the callback:

```lua
MCM.EventButton.RegisterCallback("reset_button", function()
    MyMod:Reset()
end)
```

## List settings

Define a list in your blueprint:

```json
{
  "Id": "ignore_list",
  "Name": "Ignore List",
  "Type": "list_v2",
  "Default": {
    "Enabled": true,
    "Elements": []
  },
  "Options": {
    "ShowSearchBar": true,
    "PageSize": 10
  },
  "Tooltip": "List of items to ignore"
}
```

Work with list items:

```lua
-- Get enabled items
local enabledItems = MCM.List.GetEnabled("ignore_list")

-- Check if specific item is enabled
local isIgnored = MCM.List.IsEnabled("ignore_list", "SomeItem")

-- Set item enabled state
MCM.List.SetEnabled("ignore_list", "SomeItem", true)

-- Add suggestions
MCM.List.InsertSuggestions("ignore_list", {"Item1", "Item2", "Item3"})
```

## Best practices

<CardGrid>
  <Card title="Use descriptive IDs" icon="setting">
    Keep setting IDs short but descriptive. They are used programmatically.
  </Card>

<Card title="Use 'Options'" icon="setting">
  Almost every setting type has `Options` that can be set. Play around with these.
</Card>

<Card title="Subscribe to events" icon="rocket">
  Use `MCM_Setting_Saved` events to react to changes in real-time.
</Card>

  <Card title="Test all widgets" icon="warning">
    Test your blueprint thoroughly - MCM validates but you should verify the UX.
  </Card>
</CardGrid>
