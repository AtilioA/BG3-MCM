# Welcome to MCM

<Aside icon="discord" type="note" title="Discord community">
  Join Volitio's{' '}<a href="https://discord.gg/DcS8c7KUa6" target="_blank">Discord server!</a>
</Aside>

**Mod Configuration Menu (MCM)** is an in-game framework for **Baldur's Gate 3** that provides a centralized UI for managing mod settings and keybindings. Created by [Volitio](https://next.nexusmods.com/profile/Volitio), MCM eliminates the need for manual JSON edits, offering a seamless experience for both mod authors and players.

<Aside type="tip" title="Why MCM?">
  Traditionally, BG3SE mods requiring settings needed manual JSON editing or custom UI
  implementations. MCM provides a unified, user-friendly interface so mod authors can focus on their
  mod's features rather than configuration boilerplate.
</Aside>

## What MCM provides

### 15+ widget types

MCM supports a wide variety of input types:

- **Booleans**: `checkbox`
- **Numbers**: `int`, `float`, `slider_int`, `slider_float`, `drag_int`, `drag_float`
- **Text**: `text` (single-line and multiline)
- **Choices**: `enum` (dropdown), `radio` (radio buttons)
- **Lists**: `list_v2` (with search, reordering, enable/disable)
- **Colors**: `color_picker` (visual picker), `color_edit` (RGBA fields)
- **Input**: `keybinding_v2` (hotkey configuration)
- **Actions**: `event_button` (buttons that trigger callbacks)

### Real-time updates

Settings changes take effect immediately without requiring save reloads.

### Keybinding management

Define keybindings in your blueprint and MCM handles:

- UI for configuring keybindings
- Loading and saving preferences
- Conflict detection and resolution
- Callback registration

### Multiple profiles

Create, save, load, and delete configuration profiles - useful for testing different settings combinations.

### Localization support

Provide translations for different languages via handle references in your blueprint.

## How it works

MCM acts as a bridge between your mod and the game UI:

1. **Define a blueprint** (`MCM_blueprint.json`) describing your settings structure
2. **MCM validates** your blueprint and generates the UI automatically
3. **Players configure** settings in the MCM UI (accessible via ESC menu or INSERT key)
4. **Your mod reads** settings via `MCM.Get("settingId")` anywhere in your code

## What you can configure

With MCM, you can expose any configurable aspect of your mod:

- **Feature toggles** (checkboxes)
- **Numeric parameters** (sliders, draggers)
- **Mode selection** (dropdowns, radio buttons)
- **Text input** (names, patterns)
- **Color schemes** (color pickers)
- **Keyboard shortcuts** (keybindings)
- **Lists** (allow/deny lists, presets)

## Quick example

<Tabs>
  <TabItem label="Blueprint">
    ```json
    {
        "SchemaVersion": 1,
        "ModName": "My Awesome Mod",
        "Tabs": [
            {
                "TabId": "settings",
                "TabName": "Settings",
                "Settings": [
                    {
                        "Id": "enable_mod",
                        "Name": "Enable Mod",
                        "Type": "checkbox",
                        "Default": true,
                        "Tooltip": "Master toggle for the mod"
                    },
                    {
                        "Id": "mode",
                        "Name": "Operating Mode",
                        "Type": "enum",
                        "Default": "Normal",
                        "Options": {
                            "Choices": ["Normal", "Enhanced", "Experimental"]
                        },
                        "Tooltip": "Select operating mode"
                    }
                ]
            }
        ]
    }
    ```
  </TabItem>

  <TabItem label="Lua code">
    ```lua
    -- Get setting values
    local enabled = MCM.Get("enable_mod")
    local mode = MCM.Get("mode")

    -- Listen to changes
    Ext.ModEvents.BG3MCM["MCM_Setting_Saved"]:Subscribe(function(payload)
        if payload.settingId == "enable_mod" then
            MyMod:SetEnabled(payload.value)
        end
    end)

    -- Set a value programmatically
    MCM.Set("enable_mod", false)
    ```

  </TabItem>
</Tabs>

## Requirements

<Steps>
  1. **Baldur's Gate 3** (with latest patch)
  2. **BG3 Script Extender (BG3SE)** - required for MCM to function
  3. **MCM mod** installed ([Nexus Mods](https://www.nexusmods.com/baldursgate3/mods/9162))
  4. **Basic Lua knowledge** (for BG3SE mod development)
</Steps>
