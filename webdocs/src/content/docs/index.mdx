## Why Use MCM?

<CardGrid>
  <Card title="No Boilerplate" icon="rocket">
    MCM provides a complete configuration system so you don't have to write JSON handling, UI code, or persistence logic. **18k+ lines of code** written for you.
  </Card>

  <Card title="No IMGUI Required" icon="laptop">
    Define settings in a simple JSON blueprint and MCM generates the UI automatically. No client-side IMGUI code needed.
  </Card>

  <Card title="Validation Built-In" icon="information">
    MCM runs dozens of validation checks on your blueprint and provides detailed error messages if something is wrong.
  </Card>

  <Card title="Event-Driven" icon="setting">
    Subscribe to setting changes via the `MCM_Setting_Saved` event. React to changes in real-time without polling.
  </Card>
</CardGrid>

</div>

## Quick Example

See how easy it is to add a settings UI to your BG3SE mod:

<Code code={`{
    "SchemaVersion": 1,
    "ModName": "My Awesome Mod",
    "Tabs": [
        {
            "TabId": "general",
            "TabName": "General",
            "Settings": [
                {
                    "Id": "enable_feature",
                    "Name": "Enable Feature",
                    "Type": "checkbox",
                    "Default": true,
                    "Tooltip": "Toggle the main feature on/off"
                },
                {
                    "Id": "intensity",
                    "Name": "Intensity",
                    "Type": "slider_int",
                    "Default": 50,
                    "Options": { "Min": 0, "Max": 100 },
                    "Description": "Adjust the intensity level"
                }
            ]
        }
    ]
}`} lang="json" title="MCM_blueprint.json" />

<Code code={`-- In your mod's Lua code
local enabled = MCM.Get("enable_feature")
local intensity = MCM.Get("intensity")

-- Listen to setting changes
Ext.ModEvents.BG3MCM["MCM_Setting_Saved"]:Subscribe(function(payload)
    if payload.settingId == "enable_feature" then
        MyMod:SetEnabled(payload.value)
    end
end)`} lang="lua" title="Using MCM in your mod" />
